When you are ready to collect payment, OpenPOS provides a simple checkout flow. The order is confirmed through a browser dialog, the running total is displayed, and the order is cleared so you can start the next one immediately.

## Checkout and clear buttons

The Order Summary footer shows two buttons whenever there is at least one item in the order:

- **Checkout** — processes the current order, shows a confirmation dialog, and clears the order
- **Clear** — removes all items from the order without processing a checkout

Both buttons are hidden when the order is empty and become active as soon as an item is added.

## How to complete a checkout

<Steps>
  <Step title="Build the order">
    Add items to the order using the menu panel buttons. The **Checkout** button appears in the Order Summary footer as soon as the first item is added.
  </Step>
  <Step title="Review the Order Summary">
    Confirm that all items, quantities, and the running total in the footer are correct before proceeding.
  </Step>
  <Step title="Click Checkout">
    Click the **Checkout** button. A browser alert dialog appears with the current date, the order number, the total amount due, and a payment confirmation message.
  </Step>
  <Step title="Confirm the dialog">
    Click **OK** in the browser alert to confirm. The order is cleared and the session order counter increments by 1.
  </Step>
</Steps>

<Note>
  The checkout confirmation is a native browser `alert()` dialog. It cannot be customized and must be dismissed with the **OK** button before you can interact with the POS again.
</Note>

## The checkout function

```javascript
$scope.checkout = function (index) {
    alert($scope.getDate() + " - Order Number: " + ($scope.totOrders + 1) + "\n\nOrder amount: $" + $scope.getTotal().toFixed(2) + "\n\nPayment received. Thanks.");
    $scope.order = [];
    $scope.totOrders += 1;
};
```

The alert message is assembled from three pieces:

- **Date** — today's date formatted as `dd/mm/yyyy`
- **Order number** — the current session order counter plus 1
- **Order amount** — the result of `getTotal()`, formatted to two decimal places

After the alert is dismissed, `$scope.order` is set to an empty array and `$scope.totOrders` is incremented.

## Date formatting

The date shown in the confirmation dialog is generated by `getDate`:

```javascript
$scope.getDate = function () {
    var today = new Date();
    var mm = today.getMonth() + 1;
    var dd = today.getDate();
    var yyyy = today.getFullYear();

    var date = dd + "/" + mm + "/" + yyyy

    return date
};
```

Dates are formatted as `dd/mm/yyyy` — for example, `19/4/2026`. Note that day and month values are not zero-padded.

## Clearing the order without checkout

Click **Clear** to remove all items from the current order without triggering a confirmation dialog or incrementing the order counter.

```javascript
$scope.clearOrder = function () {
    $scope.order = [];
};
```

Use **Clear** when you need to start over on the current order — for example, if a customer changes their mind before payment is taken.

<Info>
  Order data is not persisted after checkout. OpenPOS is session-based for orders: once the order is cleared (either by completing checkout or clicking **Clear**), it cannot be recovered. Only your product catalog is stored in MongoDB and persists across sessions.
</Info>
