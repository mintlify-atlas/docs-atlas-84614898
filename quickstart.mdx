---
title: "Quickstart: install and run OpenPOS in minutes"
description: "Clone the repository, install dependencies, configure your database connection, and have OpenPOS running in your browser in a few minutes."
---

This guide walks you through everything you need to go from zero to a running OpenPOS instance. By the end, you'll have a local server accepting connections and a user account ready to log in.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js and npm** — download from [nodejs.org](https://nodejs.org/) for your platform.
- **MongoDB** — either a local installation ([MongoDB installation docs](https://docs.mongodb.com/manual/installation/)) or a free [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) cluster.

## Setup steps

<Steps>
  <Step title="Clone the repository">
    Clone the OpenPOS repository from GitHub and navigate into the project directory.

    ```bash
    git clone https://github.com/kimdj/OpenPOS.git
    cd ./OpenPOS
    ```
  </Step>

  <Step title="Install dependencies">
    Install all Node.js dependencies declared in `package.json`.

    ```bash
    npm install
    ```

    This installs the full dependency tree, including Express, Mongoose, Passport.js, Gulp, and Browsersync.
  </Step>

  <Step title="Configure your database connection">
    OpenPOS supports two database options. Open `server.js` and update the connection settings before starting the server.

    See the [configuration guide](/configuration) for detailed instructions on switching between a local MongoDB instance and MongoDB Atlas.
  </Step>

  <Step title="Start the server">
    You have two options for starting OpenPOS.

    <CodeGroup>

    ```bash gulp (recommended)
    gulp
    ```

    ```bash node
    node server.js
    ```

    </CodeGroup>

    <Tip>
      Running `gulp` starts the server with nodemon (auto-restarts on file changes) and spins up a Browsersync proxy on port 3010 that reloads all connected browsers whenever `.handlebars` view files change. Use `node server.js` for a plain server without live reload.
    </Tip>
  </Step>

  <Step title="Open the app in your browser">
    Once the server is running, open your browser to one of the following addresses:

    <Note>
      The default server port is **3000**. When you run `gulp`, Browsersync also starts a proxy on port **3010** that provides live reload during development.
    </Note>

    | Method | URL |
    |---|---|
    | Direct (node or gulp) | http://localhost:3000 |
    | Browsersync proxy (gulp only) | http://localhost:3010 |
  </Step>

  <Step title="Register an account and log in">
    On your first visit, navigate to the **Register** page to create a new user account. Once registered, log in with your credentials.

    Each account has its own isolated product catalog stored in MongoDB — products you add are only visible to your account.
  </Step>
</Steps>
